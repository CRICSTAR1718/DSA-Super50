How to use this plan
01
The 10-min rule
Spend the first 10 minutes only identifying the pattern — don't code yet. Write it down. Trains recognition independently of implementation.
02
3-attempt system
Attempt 1: cold. Attempt 2: after re-reading constraints. Attempt 3: after pattern reveal. Track which attempt you needed.
03
Template library
After each week, write a 15–20 line code template for that pattern. By week 12 you have a personal reference to skim before any contest.
04
Sunday review
Re-read only the problem statements of the week. Can you identify each pattern in under 60 seconds? That's your progress check.

Pattern triggers
Sliding Window → "contiguous subarray/substring", "window of size k".

Two Pointers → sorted array + find pair, palindrome check, or merging two sequences.
Key distinction
Sliding Window keeps a range [l, r] expanding right. Two Pointers moves both ends toward each other. When you can sort the input first, Two Pointers almost always applies.

Pattern triggers
"Minimize the maximum" / "maximize the minimum" → binary search on the answer.

Write a feasibility function first. If feasible(mid) is monotone, binary search applies.
Template skeleton
lo, hi = min_val, max_val
while lo < hi:
  mid = (lo+hi)//2
  if feasible(mid):
    hi = mid
  else:
    lo = mid+1
return lo

Pattern triggers
Greedy → "maximize/minimize" + single scan feels sufficient + no overlapping subproblems.

Intervals → sort by end for activity selection; sort by start for merging.
Greedy vs DP test
Ask: "does my choice now ever hurt a future choice?" If yes → you need DP. If no → Greedy works. When unsure, try greedy first, then verify with a small counterexample.

Pattern triggers
Both children needed → post-order DFS.
Level-by-level → BFS with queue.
Tracking path from root → pre-order DFS, pass state down.
DFS return trick
For problems like diameter or max path sum, the function returns the "best single branch" upward but updates a global max using both branches. This is the key insight for 80% of hard tree problems.

Union Find template
parent[i] = i # init
def find(x):
  if parent[x]!=x:
    parent[x]=find(parent[x])
  return parent[x]
def union(x,y):
  parent[find(x)]=find(y)

Two-heap trick
Median of stream uses a max-heap (left half) and min-heap (right half). The median is always at the tops. Rebalance after every insert to keep sizes ≤1 apart.

Pattern triggers
"Generate all / find all valid X" → Backtracking.

Template: for each choice → if valid → recurse → undo. Distinct from DP: backtracking explores all paths; DP caches repeated subproblems.
Pruning levels
Light pruning (subsets) → skip duplicates. Medium (generate parentheses) → check counts. Heavy (N-Queens, Sudoku) → check row/col/box at each step. The harder the constraint, the more you can prune.

This week's drill
Write the pattern name on paper before opening the editor. Compare with editorial intent. Track your hit rate — aim for ≥75% by end of week. Every miss goes on your weak-patterns list.
Common confusion pairs
  • Greedy vs DP → try greedy, find counterexample
  • Backtracking vs DP → backtrack = all paths, DP = cached subproblems
  • BFS vs Dijkstra → BFS for unweighted, Dijkstra for weighted
  • Union Find vs DFS → UF for online queries, DFS for one-shot
