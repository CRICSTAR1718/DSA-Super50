How to use this plan
🎯 The 10-min rule
Spend the first 10 minutes ONLY identifying the pattern — don't code yet. Write it down. This trains recognition independently of implementation.
🔁 3-attempt system
Attempt 1: cold. Attempt 2: after re-reading constraints. Attempt 3: after pattern reveal. Track which attempt you needed — that's your data.
📋 Template library
After each week, write a 15–20 line code template for that pattern. By week 12 you have a personal reference to skim before any contest.
📅 Sunday review
Every Sunday, re-read only the problem statements of the week. Can you identify each pattern in under 60 seconds? That's your progress check.

Final review checklist
